HPI Helpers

The HPI Helpers are a set of classes that can be used by the HPI test cases. To put it simply, they provide for code re-use.

AlarmHelper
Contains a set of static methods for working with the SaHpiAlarmT structure as well as modifying and querying the DAT.
AlarmSet
Represents a set of alarms.
AnnouncementSet
Represents a set of Annunciator announcements.
AnnunciatorHelper
Contains a set of static methods for working with the SaHpiAnnunciatorRecT and SaHpiAnnouncementT structures.
CharBuffer
This class makes simplifies the handling of characters. In HPI, characters are represented by bytes. The CharBuffer class converts those bytes to characters depending upon the data type: Unicode, Text, etc. To put it simply, working with the actual characters is simpler than working with a byte stream. It is also possible to convert a CharBuffer to a SaHpiTextBufferT for use with the HPI functions.
ConditionHelper
Contains a set of static methods for working with the SaHpiConditionT structure.
ControlHelper
Contains a set of static methods for working with the SaHpiCtrlRecT structure.
EventHandler
The EventHandler class acts as an interface. Derived classes that extend EventHandler are able to register for callbacks for various types of event handling.
EventHelper
Contains a set of static methods for working with HPI events. Some of the methods allow for manipulation of the SaHpiEventT structure. Others allow for the waiting of events, generating of events, and clearing of the event queue.
EventLogHelper
Contains a set of static methods for working the an Event Log. Methods are availble for determining if the Event Log is empty, adding entries to the log, filling the log entirely, overflowing the log, making room for adding new entries, and for finding entries in the log.
EventMatcher
The EventMatcher is used to match against events, typically those that are being retrieved from the event queue. The EventMatcher supports the following attributes: Source, EventType, Severity, HotSwapState, SensorNum, SensorAssertion, and SensorEventState. If and only if an attribute is set, then an incoming event must match the values for those attributes that are set.
HotSwapHelper
Contains a set of static methods for determining if the AutoInsertTimeout or AutoExtractTimeout values can be changed.
HpiHelper
Contains a set of generic static methods pertaining to HPI.
InventoryHelper
Contains a set of static methods for working with IDRs, Areas, and Fields.
MinorAlarmGenerator
The MinorAlarmGenerator, as its name implies, is used to generate a minor alarm. It does this by finding a sensor threshold for which an UpMinor event can be generated. A method is also provided for clearing the alarm at a later time.
NameHelper
Contains a set of static methods for working with the SaHpiNameT structure.
RdrHelper
Contains a set of static methods for working with the SaHpiRdrT structure.
ResourceHelper
Contains a set of static methods for working with RPT entries, entity paths, and resource info.
SensorHelper
Contains a set of static methods for working with sensors. This includes sensor records, readings, thresholds, etc.
TextBufferHelper
Contains a set of static methods for working with Text Buffers.
UpMinorEventGenerator

The UpMinorEventGenerator is, as its name implies, used to generate an UpMinor event. The sensor's UpMinor threshold value is changed to be equal to its LowMinor value. Since the sensor's current reading will be between the LowMinor and UpMinor values, the reading is now above the UpMinor. This should normally cause the UpMinor event to be generated. The following is the typical sequence of steps for using this class.

  1. Create an instance of UpMinorEventGenerator for a specific sensor.
  2. Use the method canAssertEvent() to determine if an event can be generated for this sensor. This might result in the sensor's settings being modified, i.e. a disabled sensor may become enabled.
  3. Optionally, invoke one of the methods to disable the sensor, event generation, or remove an event mask. These methods are only used by test cases that verify that the event does not get generated.
  4. Invoke assertEvent() to cause the event (or non-event) to occur.
  5. Invoke deassertEvent() to clear the event.
  6. Invoke restoreSensor() to restore the sensor to its original values, i.e. if the sensor was originally disabled, it will be disabled again.
WatchdogHelper
Contains a set of static methods for working with watchdog data structures.
